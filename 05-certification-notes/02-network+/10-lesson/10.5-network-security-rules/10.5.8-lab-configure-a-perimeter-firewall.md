## Objective
Configure firewall rules in pfSense to:
- Allow HTTP and HTTPS traffic from the WAN to a web server in the DMZ.
- Allow all traffic from the LAN to the DMZ.

## ✅ Task Checklist

### 1. 🔥 Rule: Allow HTTP from WAN → DMZ Web Server

- Navigate: `Firewall > Rules > DMZ`
- Click `Add`
- Set:
  - **Action**: Pass
  - **Source**: `WAN net`
  - **Destination**: `Single host or alias`
  - **Destination IP**: `172.16.1.5`
  - **Port Range**: HTTP (80)
  - **Description**: `HTTP from WAN to DMZ`
- Click **Save** > **Apply Changes**

---

### 2. 🔒 Rule: Allow HTTPS from WAN → DMZ Web Server

- Copy the previous rule using the **Copy** icon
- Change:
  - **Port Range**: HTTPS (443)
  - **Description**: `HTTPS from WAN to DMZ`
- Click **Save** > **Apply Changes**

---

### 3. 🔁 Rule: Allow All from LAN → DMZ

- Click `Add` (in `Firewall > Rules > DMZ`)
- Set:
  - **Action**: Pass
  - **Protocol**: Any
  - **Source**: `LAN net`
  - **Destination**: `DMZ net`
  - **Description**: `LAN to DMZ Any`
- Click **Save** > **Apply Changes**

---

## 🧠 Notes & Best Practices

- **WAN net** includes all IPs on the external side of the firewall.
- Rules are processed **top to bottom** – order matters.
- Always use descriptive labels in rule descriptions.
- Don’t forget to hit **Apply Changes** after saving rules.
