# Section 13.1
- **WAN vs LAN** â†’ WAN covers greater distances, often uses public telecom infrastructure.  
- **Physical Layer (WAN)** â†’ Often uses modems for modulation/demodulation; can be analog or digital.  
- **Data Link Layer (WAN)** â†’ Simpler protocols than LAN; often point-to-point.  
- **Network Layer (WAN)** â†’ CE router connects customer network to provider; provider assigns IP addresses.  
- **Public networks** â†’ Chosen mainly for cost savings over private solutions.  
- **T-carrier (T1)** â†’ Uses Time Division Multiplexing (TDM) to carry multiple channels on one line.  
- **DSL** â†’ Uses phone lines; requires splitters to avoid interference; ADSL = asymmetrical, SDSL = symmetrical.  
- **Cable Internet** â†’ Uses coax + DOCSIS standard; connects through CMTS.  
- **FTTC** â†’ Fiber to cabinet, copper last segment; usually uses VDSL.  
- **FTTP** â†’ Fiber directly to premises; uses Optical Network Terminal (ONT) to convert optical to Ethernet.
# Section 13.2
- **13.2.1 Remote Access Considerations** â†’ Remote access = indirect network connection (often VPN). Needs authentication, access restrictions, logging, and up-to-date hardware/software.
- **13.2.2 Tunneling Protocols** â†’ Tunneling wraps private packets inside public ones. PPP (Layer 2, no security), GRE (Layer 3, no auth), IPSec (secure, Layer 3), TLS/DTLS (secure but more overhead).
- **13.2.3 IPSec** â†’ AH = integrity/authentication only, ESP = encryption + integrity. Two modes: Transport (payload only) and Tunnel (entire packet).
- **13.2.4 IKE** â†’ Negotiates IPSec security settings and keys. Phase I = secure channel + authentication. Phase II = sets ciphers/keys. IKEv2 adds mobility, NAT traversal, EAP support.
- **13.2.5 Client-to-Site VPNs** â†’ Remote workers connect to corporate gateway. Uses SSTP, L2TP/IPSec, or SSL/TLS. Split tunnel = faster but less secure; full tunnel = secure but slower.
- **13.2.6 Clientless VPNs** â†’ No client software; works via browser (HTML5, WebSockets). Often used for remote desktop or app access.
- **13.2.7 Site-to-Site VPNs** â†’ Connects whole networks via gateways. Can be static (always on) or dynamic (on demand). Works over internet or private WAN; also for local segmentation.
